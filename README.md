Linux Kernel Module: ex_queue

Лабораторная работа по изучению структур данных в ядре Linux.

Этот проект представляет собой простой модуль ядра (*.ko), который демонстрирует работу с очередью (FIFO) в пространстве ядра. Модуль использует стандартный API kfifo для создания очереди сообщений и таймеры для имитации асинхронной обработки данных.
Описание функционала

При загрузке модуль выполняет следующие действия:

    Инициализирует очередь фиксированного размера.
    Добавляет в очередь 5 сообщений с текстами Message #1 ... Message #5.
    Запускает таймер, который каждые 2 секунды извлекает одно сообщение из очереди и выводит его содержимое в системный лог (dmesg).
    Процесс продолжается, пока очередь не опустеет.

При выгрузке модуль безопасно останавливает таймер и очищает очередь от оставшихся элементов.

Все операции (добавление, обработка, удаление) логируются, что наглядно демонстрирует принцип работы FIFO (First In, First Out).
Требования

Для сборки и запуска модуля вам понадобится:

    ОС: Ubuntu (или другой дистрибутив Linux).
    Компоненты: Установленные заголовки ядра и инструменты сборки.

Установить зависимости можно одной командой:

sudo apt-get update && sudo apt-get install -y build-essential linux-headers-$(uname -r)

Инструкция по использованию

Весь процесс сборки, загрузки и проверки автоматизирован с помощью Makefile.
1. Сборка модуля

Перейдите в папку проекта и выполните:

make

Или просто:

make all

2. Быстрая проверка (Сборка -> Загрузка -> Лог)

Эта команда соберет модуль, загрузит его в ядро, подождет около 12 секунд (чтобы вы увидели обработку всех 5 сообщений), выгрузит модуль и покажет итоговый лог.

sudo make check

(Примечание: make check включает задержку на 12 секунд для наглядности).
3. Ручное управление

    Загрузить модуль в ядро:

    sudo make run

    Выгрузить модуль из ядра:

    sudo make remove

    Посмотреть лог ядра (вывод printk):

    dmesg | tail -n 40

4. Очистка проекта

Удаляет все скомпилированные файлы (*.o, *.ko, *.mod.*):

make clean

5. Форматирование кода

Приводит код к единому стилю с помощью clang-format:

make format

Важные примечания

    Ошибки компиляции: Убедитесь, что версия компилятора gcc совпадает с версией, использованной для сборки вашего ядра. Обычно это происходит автоматически при установке пакета linux-headers.
    Ошибки загрузки (Invalid argument): Если вы используете цель install или uninstall, убедитесь, что версия ядра (uname -r) не изменилась между командами. Для учебных целей рекомендуется использовать make run и make remove.
    Безопасность: Работа в пространстве ядра может привести к сбою всей системы (Kernel Panic). Не запускайте модуль на критически важных серверах или рабочих станциях с несохраненными данными.

Как проверить результат?

После выполнения sudo make check или ручного запуска (run) и просмотра лога (dmesg), вы должны увидеть следующую последовательность:

    Сообщения о добавлении элементов в очередь (Added to queue).
    Через паузы в 2 секунды — сообщения об обработке (Processed msg id=...).
    Порядок обработки будет строго соответствовать порядку добавления: сначала id=1, затем id=2 и так далее. Это и есть демонстрация работы очереди FIFO.
